# Changelog

## UX Fix Pack

- **Fix 19 — Filter dialog person list is now a `LazyColumn`** — Replaced `filteredPersonList.forEach { Row(...) }` inside `QuizFilterDialog` with a `LazyColumn(modifier = Modifier.heightIn(max = 280.dp), verticalArrangement = spacedBy(2.dp))` using `items(key = { it.id })`. The bounded height prevents the dialog from growing unboundedly with large user lists and lets the person list scroll independently within the 280dp cap while the outer `verticalScroll` column handles the rest of the dialog.
- **Fix 18 — Sign-in flow regressions + minor cleanups** — 18a: `PickUserScreen` now signs in directly for uniquely-named users; confirmation dialog only appears when the display name contains a `(N)` disambiguation suffix. 18b: "Not you?" and "Switch User" now fire separate dialogs with context-appropriate copy ("Sign out ${name}? The next person can sign in from the picker." vs. "Switch user? You can sign back in from the picker."). 18c: `DuplicateNameDialog` dismiss button changed from "Add a middle initial or more" to "No, I'm different"; added a hint line explaining how to disambiguate. Minor cleanups: `ActiveUsersScreen` "Filter by Group" label routed through `t()`; "Still setting up…" now replaces (not appends to) the stats line for ineligible users; filter dialog Spanish "N seleccionadas" now uses the `plural` helper for correct singular ("1 seleccionada"); `HomeViewModel.renameUser` replaced `transitionToUserSelected` with a targeted `_uiState.value = cur.copy(user = updatedUser, renameError = null)` to avoid restarting collect jobs.
- **Fix 17 — Idle timeout is now route-aware** — `IdleTimeout` signature changed from `timeoutMs: Long` to `timeoutMs: () -> Long` (lambda, read on every check via `rememberUpdatedState`). `MainActivity` derives the timeout from `currentBackStackEntryAsState()`: survey and quiz routes get 3 minutes, admin gets 5 minutes, everything else stays at 90 seconds. Also de-flaked `QuizSessionTest.distribution_*`: added an optional `rng: Random` parameter to `buildQuizSessionFromPools` so the test can pass seeded randoms; the test now aggregates 10 seeded 30-pick runs and checks the 300-pick aggregate falls 40–80 per subject instead of asserting a single random 30-pick run.
- **Fix 16 — Filter dialog pool-size no longer thrashes the DB on every chip tap** — Added `loadAllSubjectPools(quizTakerId)` to `GTKYRepository` which computes per-subject available-question counts in one pass (eligibility check + already-attempted filter). Added `getGroupMembersMap(groupIds)` for a thin DAO lookup and `filterSubjectPools(pools, groupIds, subjectUserIds, groupMembers)` for pure in-memory filtering using the same logic as `resolveEligibleSubjects`. `HomeViewModel` stores `_allSubjectPools` and `_groupMembersMap`, loading them once on `transitionToUserSelected` and refreshing on the `observeTotalAnswers()` signal. `updateFilterPreview` now does zero DB work — it filters the cached list and sums `availableCount`, capped at 30.
- **Fix 14 — "Take a Quiz" buttons in Survey and Connections actually open the quiz filter** — Added `requestOpenQuizDialog()` / `clearPendingOpenQuizDialog()` to `HomeViewModel` (mirrors the existing `requestQuizWithSubject` pattern). `UserHomeScreen` gains a second `LaunchedEffect(pendingOpenQuizDialog)` that sets `showQuizFilterDialog = true` and clears the flag. `SurveyScreen` and `ConnectionsScreen` each gain an `onGoToQuiz: () -> Unit = {}` param; `NavGraph` wires both by grabbing the Home back-stack `HomeViewModel`, calling `requestOpenQuizDialog()`, then `popBackStack(HOME)`. "Take a Quiz" in `SurveyScreen.AllAnsweredContent` and in the Connections MINE empty state now route through those callbacks instead of `onBack`.
- **Fix 13 — Quizzable-users flow now re-emits in real time** — Added `getTotalAnswerCountFlow()` to `SurveyAnswerDao` (flows off the whole `survey_answers` table). `getQuizzableUsers` and the home screen's ready-count now both react when *any* user answers questions, not just the current user. The quiz filter dialog's person list and the "Quiz unlocks when…" subtitle update live without the current user doing anything.
- **Fix 12 — Group eligibility threshold corrected** — Removed `getQuizEligibleUsersInGroup` from `UserDao`, which still hardcoded the old 15-answer threshold. Group-filtered quizzes now use the same `QUIZ_UNLOCK_THRESHOLD` (8) as all other paths.
- **Fix 11 — Quiz subject distribution and multi-select** — Quiz sessions now interleave subjects using weighted random, biased toward people you've been quizzed on less. Fixes clustering where one or two people dominated a whole session. New unified quiz filter dialog: filter by group, by specific people (with search), or both. Selecting specific people overrides the group filter. Active Users tap now opens the filter dialog with the person pre-selected, so you can add more people before starting. Pre-start pool-size warning tells you how many questions are available for your filter.

- **Fix 10 — Small cuts** — Added `plural(count, singEn, plurEn, singEs, plurEs)` @Composable helper to `LocalAppLanguage.kt`. Used it to fix `AdminScreen`'s English-only `"${n} answers"` label (now bilingual), and to simplify `HomeScreen`'s "You've answered N question(s)" ternary. `WelcomeScreen` LanguageToggle moved from a floating `Box(align=TopEnd)` overlay to a `Row(Arrangement.End)` as the first child of an outer Column — same visual, proper layout hierarchy. `HomeUiState.PickGroups` branch now renders a plain `Box` placeholder behind `GroupPickerDialog` instead of the full `WelcomeScreen`, eliminating the form flash. `QuizGroupPickerDialog` no longer silently flips back to "All Groups" when the last group chip is deselected; instead the "Start Quiz" button stays disabled and a red `"Pick at least one group"` error line appears. `GroupPickerDialog` (signup group picker) gains a hint line below the list: `"You can join groups later from the Groups screen."`
- **Fix 9 — Admin PIN first-run forced change** — `DataSeeder` now seeds `AppConfig("admin_pin_is_default", "true")` alongside the default `"1234"` PIN. `GTKYRepository.setAdminPin` clears the flag via `deleteKey` on every call, so it's erased as soon as any deliberate change is made. `GTKYRepository.getAdminPinIsDefault()` reads the flag. `AdminViewModel` loads `isPinDefault` on `init` and re-reads it after a successful `authenticate`; on auth with flag set, `mustChangePin = true` is added to `AdminUiState`. `AdminScreen` shows a non-dismissible `ChangePinDialog` (no Cancel button, `onDismissRequest = {}`) immediately when `isAuthenticated && mustChangePin`; successful PIN change sets `mustChangePin = false` in the same update. `PinEntryScreen` gains an `isPinDefault` param and shows a hint below the subtitle: "First time? Default PIN is 1234. You'll be asked to change it."
- **Fix 8 — Active Users screen becomes a leaderboard you can act on** — Added `buildQuizSessionForSubject(quizTakerId, subjectUserId, count)` to `GTKYRepository`. Extended quiz route to `"quiz/{userId}/{groupIds}?subjectId={subjectId}"` with `defaultValue = -1L`; `Routes.quizSubject(quizTakerId, subjectId)` helper navigates with `groupIds = "0"`. `QuizViewModel` accepts `subjectId: Long = -1L` in constructor and factory; when set, `loadQuiz` calls `buildQuizSessionForSubject` instead of the group-based path. Added `noQuestionsForSubject: Boolean` + `subjectDisplayName: String?` to `QuizUiState`; `QuizScreen` shows new `AcedSubjectContent` ("You've already aced every question about $name!") when the subject-specific session comes back empty. `ActiveUsersScreen` gains `onStartSubjectQuiz: (Long) -> Unit` callback; eligible non-current-user rows are now `clickable` and open a confirmation `AlertDialog`; non-eligible rows show a "Still setting up…" sub-label. `NavGraph` wires `onStartSubjectQuiz` with a guard on `activeUserId`.
- **Fix 7 — Survey question grammar transformer** — Created `util/QuestionPhrasing.kt` with `forSurvey(template, lang)` (delegates to `QuestionUtils.toSelfEn/toSelfEs`) and `forQuiz(template, subjectName)`. `SurveyScreen` now calls `forSurvey` instead of inline `QuestionUtils` calls. `QuizScreen` routes through `forQuiz` for symmetry. 22 unit tests in `QuestionPhrasingTest` cover 10 EN + 10 ES survey transforms + 2 quiz passthroughs. Added JUnit 4.13.2 `testImplementation` to build.
- **Fix 6 — Quiz flow polish** — TopAppBar title now reads "Quiz — about $name". "Q X/Y" progress counter in actions replaces the old "Remaining: N" / "Keep Going!" text. `LinearProgressIndicator` added under the bar; "Keep Going!" reappears as a right-aligned badge below the bar once `canFinish` is true. Correct answers auto-advance after 1400 ms via `LaunchedEffect`; "Next →" affordance lets users skip the wait. Wrong answers keep the explicit "Next Question" button with no auto-advance. "I haven't met them" skip now shows a 1.5 s "Skipped $name" toast card (`justSkippedPerson` flag in `QuizUiState`, cleared by `clearSkipToast()`). Quiz group picker dialog now shows "$N people ready to quiz about" + first-3-names preview, powered by `HomeViewModel.readyUsersByGroup`. Also fixed the hardcoded `>= 15` in `buildQuizSession` group-filtered path to use `QUIZ_UNLOCK_THRESHOLD`.
- **Fix 5 — Name collisions and identity recovery** — `createAndSelectUser` now enters `DuplicateName` state on collision instead of a generic error; Welcome screen shows "Are you them?" dialog with "Yes, that's me" (signs in as existing user) and "Add a middle initial or more" (returns to form with last-name field pre-focused). Picker rows disambiguate same-name users with "(1)", "(2)" suffix (computed at render time, no DB change). Confirmation dialog before sign-in from picker. Edit icon (pencil) added to "Signed in as" pill; `AlertDialog` with `OutlinedTextField` lets the active user rename themselves (validates uniqueness, shows inline error).

- **Fix 4 — Dead-end empty states all get primary CTAs** — Connections MINE empty: headline + body + "Take a Quiz" / "Answer more questions" buttons. Quiz no-eligible-users: new copy, optional "N players almost there" count, "Answer more questions" navigates to survey. Groups empty: added admin PIN hint + "Continue without groups" button. Survey all-answered: new body copy + "Take a Quiz" primary button.
- **Fix 3 — Idle timeout signs out the previous user** — Idle timeout shortened to 90 s. On idle, `GTKYApplication.handleIdleTimeout` clears the active user before navigating home, so the next person sees the Welcome screen. Added "Signed in as" pill at the top of the home screen with a "Not you?" button. Both "Not you?" and "Switch User" now show a confirmation dialog before signing out.
- **Fix 2 — Connections defaults to "My Connections"** — Renamed `ConnectionMode` → `ConnectionScope` (MINE/EVERYONE) + `ConnectionDirection` (MUTUAL/ONE_WAY). Connections screen now defaults to MINE scope, showing "You & $name" rows and "You know / knows you" one-way labels. Everyone view still accessible via scope toggle. MINE empty state explains how connections are earned.
- **Fix 1 — Quiz unlock threshold lowered to 8** — Introduced `Constants.QUIZ_UNLOCK_THRESHOLD = 8` and `QUIZ_MIN_QUESTIONS_BEFORE_FINISH = 5` as single sources of truth. Home screen now shows a 3-state subtitle ("Answer N more", "wait for others", or nothing). Survey now shows a `LinearProgressIndicator` and category label. Milestone toast fires when the threshold is first crossed. All six hardcoded `15` references replaced.
