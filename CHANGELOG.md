# Changelog

## UX Fix Pack

- **Fix 7 — Survey question grammar transformer** — Created `util/QuestionPhrasing.kt` with `forSurvey(template, lang)` (delegates to `QuestionUtils.toSelfEn/toSelfEs`) and `forQuiz(template, subjectName)`. `SurveyScreen` now calls `forSurvey` instead of inline `QuestionUtils` calls. `QuizScreen` routes through `forQuiz` for symmetry. 22 unit tests in `QuestionPhrasingTest` cover 10 EN + 10 ES survey transforms + 2 quiz passthroughs. Added JUnit 4.13.2 `testImplementation` to build.
- **Fix 6 — Quiz flow polish** — TopAppBar title now reads "Quiz — about $name". "Q X/Y" progress counter in actions replaces the old "Remaining: N" / "Keep Going!" text. `LinearProgressIndicator` added under the bar; "Keep Going!" reappears as a right-aligned badge below the bar once `canFinish` is true. Correct answers auto-advance after 1400 ms via `LaunchedEffect`; "Next →" affordance lets users skip the wait. Wrong answers keep the explicit "Next Question" button with no auto-advance. "I haven't met them" skip now shows a 1.5 s "Skipped $name" toast card (`justSkippedPerson` flag in `QuizUiState`, cleared by `clearSkipToast()`). Quiz group picker dialog now shows "$N people ready to quiz about" + first-3-names preview, powered by `HomeViewModel.readyUsersByGroup`. Also fixed the hardcoded `>= 15` in `buildQuizSession` group-filtered path to use `QUIZ_UNLOCK_THRESHOLD`.
- **Fix 5 — Name collisions and identity recovery** — `createAndSelectUser` now enters `DuplicateName` state on collision instead of a generic error; Welcome screen shows "Are you them?" dialog with "Yes, that's me" (signs in as existing user) and "Add a middle initial or more" (returns to form with last-name field pre-focused). Picker rows disambiguate same-name users with "(1)", "(2)" suffix (computed at render time, no DB change). Confirmation dialog before sign-in from picker. Edit icon (pencil) added to "Signed in as" pill; `AlertDialog` with `OutlinedTextField` lets the active user rename themselves (validates uniqueness, shows inline error).

- **Fix 4 — Dead-end empty states all get primary CTAs** — Connections MINE empty: headline + body + "Take a Quiz" / "Answer more questions" buttons. Quiz no-eligible-users: new copy, optional "N players almost there" count, "Answer more questions" navigates to survey. Groups empty: added admin PIN hint + "Continue without groups" button. Survey all-answered: new body copy + "Take a Quiz" primary button.
- **Fix 3 — Idle timeout signs out the previous user** — Idle timeout shortened to 90 s. On idle, `GTKYApplication.handleIdleTimeout` clears the active user before navigating home, so the next person sees the Welcome screen. Added "Signed in as" pill at the top of the home screen with a "Not you?" button. Both "Not you?" and "Switch User" now show a confirmation dialog before signing out.
- **Fix 2 — Connections defaults to "My Connections"** — Renamed `ConnectionMode` → `ConnectionScope` (MINE/EVERYONE) + `ConnectionDirection` (MUTUAL/ONE_WAY). Connections screen now defaults to MINE scope, showing "You & $name" rows and "You know / knows you" one-way labels. Everyone view still accessible via scope toggle. MINE empty state explains how connections are earned.
- **Fix 1 — Quiz unlock threshold lowered to 8** — Introduced `Constants.QUIZ_UNLOCK_THRESHOLD = 8` and `QUIZ_MIN_QUESTIONS_BEFORE_FINISH = 5` as single sources of truth. Home screen now shows a 3-state subtitle ("Answer N more", "wait for others", or nothing). Survey now shows a `LinearProgressIndicator` and category label. Milestone toast fires when the threshold is first crossed. All six hardcoded `15` references replaced.
